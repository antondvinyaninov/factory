# ADR 001: Стек проекта

## Контекст

Нужен корпоративный портал для завода с внутренними новостями, документами, ролями, поиском, RAG и возможностью развивать модули независимо.

## Решение

Используем TypeScript full-stack:

- Next.js для интерфейса;
- NestJS для API-слоя;
- PostgreSQL для основной базы;
- Redis для очередей и кэша;
- MinIO для файлов;
- Qdrant и локальную LLM для RAG;
- n8n как дополнительный workflow-инструмент, но не как центральный серверный слой.

## Последствия

- Команда может работать в одном языке: TypeScript.
- API-слой остаётся структурированным и расширяемым.
- RAG, документы и автоматизации можно развивать отдельно.
- На старте не усложняем проект микросервисами.
