# Factory 1.0

Новый проект портала вместо HumHub. Базовая идея: модульный монолит на TypeScript, который можно развивать частями без переписывания всей системы.

## Целевая архитектура

```text
apps/web  → Next.js + shadcn/ui + Tailwind
apps/api  → NestJS + TypeScript

PostgreSQL → основная база данных
Redis      → кэш, очереди, фоновые задачи
MinIO      → внутреннее файловое хранилище
Qdrant     → векторная база для RAG
Local LLM  → локальная языковая модель без внешних API
n8n        → автоматизации рядом, не центральный серверный слой
```

## Главный принцип

На старте делаем не микросервисы, а **модульный монолит**:

- одно API-приложение;
- отдельные модули внутри NestJS;
- отдельные frontend-разделы внутри Next.js;
- возможность позже вынести тяжёлые части отдельно.

## Основные модули

- `auth` — вход, сессии, токены, безопасность;
- `users` — сотрудники;
- `departments` — отделы и структура завода;
- `news` — новости и объявления;
- `documents` — документы, файлы, совместное редактирование;
- `search` — обычный поиск по порталу;
- `rag` — внутренний AI-поиск по базе знаний;
- `tasks` — заявки, поручения, задачи;
- `notifications` — уведомления;
- `audit` — журнал действий;
- `admin` — административная панель.

## Статус

Сейчас создана только структура проекта. RAG, документы и n8n пока представлены как отдельные папки-заготовки без реализации инфраструктуры.

## Деплой

Проект подготовлен к запуску одним Docker-контейнером:

- `apps/web` отдаёт интерфейс и слушает внешний порт `3000`;
- `apps/api` запускается внутри контейнера на `3001`;
- браузер ходит в API через относительный путь `/api`, без привязки к `localhost`;
- инструкция для EasyPanel лежит в `docs/deployment/easypanel.md`;
- пример production-переменных лежит в `.env.production.example`.
